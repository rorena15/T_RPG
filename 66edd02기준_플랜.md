# PROTOCOL: STIGMA — 각성 스킬 체험 + 균형 경로 힌트

## Context
두 가지 문제를 함께 해결한다:
1. 1막에서 각성 스킬을 한 번도 체험하지 못한 채 종료되는 문제
2. 황금 분할(balanced) 직업 진입 조건이 코드상 가능하나 힌트가 전무한 문제

---

## 사전 검증 결과 (구현 전 확인 완료)

### 가중치 적용 현황
- **세션 이벤트 6개**: 각 선택지 1=kinetic / 2=scrap / 3=cyber 균형 배분 ✓
- **벙커 진입 이벤트** (`녹슨 방전 벙커의 문`): 선택지 3개 모두 weight=None → 코드에서 `if choice_weight:` 분기로 안전하게 건너뜀 ✓
- **보스 코어**: 선택지 1=kinetic+3 / 2=scrap+3 / 3=cyber+3 ✓
- **균형 달성 최저 경로**: 6개 세션에서 (1:4:4) 형성 후 보스 코어에서 lagging 차원 선택 → (4:4:4) → balanced ✓

### combat_loop 시그니처 & 반환값
```python

# 반환: (hp, enemy_type) — 탈출/승리 시 정상값, 사망 시 sys.exit()
```

## 변경 1: 외부 경계 전투 (각성 스킬 체험)

### 서사 설계
- 벙커 내부에 적 재등장 불가 (안전 거점 서사 보호)
- 보스 각성 → 스킬 지급 → 벙커 외부 퍼리미터에 잔존 정찰 유닛 접근 경보 → 격퇴 or 회피 → 엔딩

### 신규 함수: `run_perimeter_encounter(player)` in `개발/Main.py`
위치: `run_ending()` 함수 정의 바로 위 (run_boss_core_choice 뒤, run_ending 앞)

```python
def run_perimeter_encounter(player):
    clear_screen()
    print_header(t('perimeter_header'))
    type_text(t('perimeter_alert_1'), 0.025)
    type_text(t('perimeter_alert_2'), 0.025)
    print()
    print_divider()
    print(t('perimeter_opt1'))
    print(t('perimeter_opt2'))
    print_divider()

    ans = read_key()
    if ans == "1":
        low_hp = int(8000 * get_turn_scale_multiplier(player) * 0.30)
        sound.play_combat_bgm()
        combat_loop(player, is_boss=False, current_hp=low_hp, enemy_type="drone")
        sound.stop_all()
        print()
        type_text(t('perimeter_win'), 0.025)
    else:
        print()
        type_text(t('perimeter_skip'), 0.025)
    time.sleep(0.8)
    wait_for_keypress()
```

※ `combat_loop` 패배 시 `sys.exit()` 직접 호출 — 별도 패배 처리 불필요
※ 탈출(escaped=True) 시에도 함수 반환되어 엔딩 정상 진행

### `run_boss_core_choice()` 수정
현재 마지막 줄 `wait_for_keypress()` 다음 줄에 추가:
```python
    run_perimeter_encounter(player)
```

## 변경 2: 균형 경로 초미세 힌트

### 설계 원칙
- 메타 설명 없음, 서사 내부 신호만
- 보스 코어 선택 화면에서 플레이어 weights가 균형 진입 조건일 때만 해당 선택지 끝에 `…` 1개 표시
- 균형 진입 조건: `(X-3 : X : X)` 형태 — 즉 한 차원이 다른 두 차원보다 정확히 3 낮음

### `run_boss_core_choice()` 수정 (locale 키 불필요, 코드 직접 처리)

```python
wk, ws, wc = player.weights["kinetic"], player.weights["scrap"], player.weights["cyber"]

# 균형 진입 조건 판별: +3 후 세 값이 동일해지는 선택지 탐지
def _balance_hint(pre_k, pre_s, pre_c):
    """보스 선택 후 balanced가 되는 선택지를 반환. 없으면 None."""
    if pre_k + 3 == pre_s == pre_c:   return "1"
    if pre_s + 3 == pre_k == pre_c:   return "2"
    if pre_c + 3 == pre_k == pre_s:   return "3"
    return None

_hint_key = _balance_hint(wk, ws, wc)

# 출력 시 해당 선택지에 dim 점 추가
opt1_suffix = f"  {Style.DIM}…{Style.RESET_ALL}" if _hint_key == "1" else ""
opt2_suffix = f"  {Style.DIM}…{Style.RESET_ALL}" if _hint_key == "2" else ""
opt3_suffix = f"  {Style.DIM}…{Style.RESET_ALL}" if _hint_key == "3" else ""

print(t('boss_core_opt1') + opt1_suffix)
...
print(t('boss_core_opt2') + opt2_suffix)
...
print(t('boss_core_opt3') + opt3_suffix)
```

### 2~4막 + DLC 확장성 평가

#### 잘 설계된 확장 포인트
- **`SESSIONS_DB`**: JSON 배열 — 2막 세션 추가만으로 새 서사 콘텐츠 투입 가능

- **장비 슬롯 11개**: DB 기반 — 새 아이템 등록만으로 확장
- **`database.json`** 단일 진실 공급원 — DLC 콘텐츠 팩을 별도 JSON으로 분리 후 병합 가능
- **`master_formulas.json`**: 수식 SSOT — 밸런스 패치 시 코드 수정 없음

#### 확장 시 주의점
