# Text RPG 프로젝트 스탯 및 핵심 연산 시스템 상세 명세서 (Core Mechanics Specification - Dual Scale Version)

본 명세서는 게임 내 모든 기초 스탯, 능력치 파생 수식, AP/SP 결합 대미지, 직업별 특수 효과 확률, 그리고 방어·저항 시스템의 수학적 모델을 정밀하게 계량화한 최종 마스터 데이터입니다. 효율 충 유저의 특정 빌드 올인을 저지하고, 하드코어 생존극의 긴장감과 라이트 유저의 초반 정착을 유기적으로 지원하도록 모든 연산 공식이 정규화되었습니다.
단 레벨이 낮은 구간부터 바로 적용 되는것이 아닌 성장 함에 따라 조금씩 점진적으로 스케일 펙터 적용을 한다

## 1. 점진적 이원화 스케일 팩터 (Progressive Display Scale)
[수정됨] 내부 연산(DMG, MaxHP)은 0.001 단위로 정밀하게 돌아가나, 유저에게 표기되는 배수는 장비의 등급(T) 개방 수준에 따라 점진적으로 진화하여 몰입감을 제어합니다.
- 1단계 [아날로그]: T=4 구역. 배수 없음. 순수 DMG 표기.
- 2단계 [전술 동기화]: T=3,2 장착 시. DMG x100, HP x10 표기.
- 3단계 [신격 스케일링]: T=1,0 장착 및 메인프레임 심부 진입 시. DMG x100,000, HP x100 표기. (이때 글리치 크러시 로그가 활성화됩니다.)

## 2. 기초 스탯 및 생체·연산 지표 파생 공식

캐릭터가 보유한 순수 기본 스탯(A_VIT, A_INT, A_DEX)과 레벨(Lv)은 플레이어의 생체 지표(오프라인 생존) 및 연산 수치(온라인 해킹)를 실시간으로 결정합니다.

### A. 체력 스탯 (A_VIT) 파생 지표

- **최대 생명력 (Max HP)** MaxHP = 100 + (Lv * 15) + (A_VIT * 20) + Math.floor(f(A_VIT) * 150)
    
- **물리 방어력 베이스 (Base DEF)** DEF_base = (Lv * 2) + (A_VIT * 1.5) + Math.floor(f(A_VIT) * 50)
    
- **인벤토리 소지 무게 한도 (kg)** WeightLimit = 30 + (A_VIT * 2.0)
    
- **생존 수치(허기/갈증) 소모 속도 감쇄율 (최대 60% 감쇄)** R_decay = min(0.60, (A_VIT * 0.01) / (1 + (A_VIT * 0.01)))
    

### B. 지력 스탯 (A_INT) 파생 지표

- **최대 사이버덱 RAM 용량 (Max RAM)** MaxRAM = 4 + Math.floor(Lv * 0.2) + Math.floor(A_INT * 0.5) + Math.floor(f(A_INT) * 10)
    
- **기초 해킹 연산 속도 가속율 (최대 +50% 제한)** B_hack = f(A_INT) * 1.0
    
- **고대 유물 감정 및 분석 성공 확률** Pr_analysis = min(0.95, 0.20 + (A_INT / (A_INT + 15)))
    

### C. 민첩 스탯 (A_DEX) 파생 지표

- **행동 턴 우선권 지수 (Initiative Speed)** INIT = 10 + (Lv * 0.5) + (A_DEX * 1.2)
    
- **치명타 확률 (Critical Rate)** CRT_rate = min(0.75, (A_DEX * 0.015) / (1 + (A_DEX * 0.015)))
    
- **기본 회피율 (Evasion Rate)** EVA_rate = min(0.50, (A_DEX * 0.01) / (1 + (A_DEX * 0.01)))
    

## 3. 효율 저지형 기본 스탯 체감 법칙 함수 f(A)

교복 빌드 및 최강 효율 스탯 독점을 완벽하게 차단하기 위한 스탯 실효 가치 변환 함수 f(A)의 구간 연산식입니다. 투자한 순수 스탯 A가 임계점을 넘을수록 가중치 효율이 계단식으로 급감합니다.

- A가 0 이상 15 이하일 때: f(A) = A * 0.02 [안정적 동기화: 효율 100%]
    
- A가 15 초과 25 이하일 때: f(A) = 0.30 + (A - 15) * 0.01 [연산 과부하: 효율 50%]
    
- A가 25 초과일 때: f(A) = 0.40 + (A - 25) * 0.002 [임계점 마비: 효율 10%]
    

## 4. AP/SP 연동형 스킬 대미지 계산 프로세스

스킬 포인트(SP)로 강화된 스킬 자체의 체급과, 능력치 포인트(AP) 투자로 변환된 기본 스탯의 실효 가치가 결합하여 최종 실대미지를 산출합니다.

### 1단계: 스킬 레벨 한계치 결정 (Soft-Cap)

MaxLevel = 20 + Math.round( (A^2) / (A + 10) )

### 2단계: 기본 스킬 위력 수치 계산 (W)

W = (SP * Alpha) * ( 1 + sum(Beta * f(A)) ) * M

- Alpha: 스킬별 고유 기본 위력 지수
    
- Beta: 스킬과 각 스탯 간의 상호작용 가중치 계수
    
- M: 마스터 보너스 가중치 (M = 1 + (SP / MaxLevel)^2 * 0.1)
    

### 3단계: 외적 평판 가중치(F) 및 패턴 노출 페널티(P) 연산

- **진영 평판 가중치 (F):** F = 1 + (R / 2000)
    
- **패턴 노출 페널티 (P):** 동일 스킬 연속 사용 횟수 E 연동
    
    - E가 10 이하일 때: P = 1.0
        
    - E가 10 초과일 때: P = max(0.5, 1.0 - (E - 10) * 0.05)
        

### 4단계: 최종 표기 실대미지 산출 (Display DMG)

DMG = max( 1, Math.floor(W * EquipPower * F * P) - DEF_eff ) **Display DMG = DMG * 100,000**

## 5. 스킬별 하이브리드 특수 효과 및 방어 · 저항 공식

두 기본 스탯 간의 투자 격차가 3 이하(|A1 - A2| <= 3)일 때 동적으로 개방되는 하이브리드 분기 연산 및 방어 수식입니다.

### A. 컴뱃 포스: [바이오 적출]

- **발동 조건:** |A_VIT - A_DEX| <= 3
    
- **적출 및 흡수 확률:** Pr_reap = 0.10 + (f(A_VIT) + f(A_DEX)) * 0.5
    
- **생존 수치 흡수량:** V_vamp = Math.floor(DMG * (0.05 + SP_Reap * 0.01))
    

### B. 메카니컬 테크: [주파수 역추적]

- **발동 조건:** |A_INT - A_DEX| <= 3
    
- **치명타 대미지 증폭 배율:** CRT_mult = 1.5 + (SP_Trace * 0.02 + f(A_DEX) * 0.5)
    
- **기습 성공 시 적 행동 불능 확률:** Pr_stun = min(0.80, (SP_Trace * 0.03) / (1 + f(A_INT)))
    

### C. 넷 포스: [프로토콜 위조]

- **발동 조건:** |A_INT - A_VIT| <= 3
    
- **전투 로그 위조 확률:** Pr_glitch = min(0.65, (SP_Glitch / MaxLevel) * f(A_INT) * 1.2)
    

### D. 최종 유효 방어력 및 저항력 공식

- **최종 유효 방어력 (DEF_eff):** DEF_eff = max( 0, (DEF_base * (1 - D_total)) - PEN ) _(※ D_total: 적용된 방어력 감소 디버프 총합. 지원형 전술 보병의 경우 f(A_INT) * 0.5 적용, 최대 50% 제한 / PEN: 방어 관통 수치)_
    
- **상태 이상 및 속성 저항력 공식 (RES):** RES = min(0.95, (f(A_stat) * EquipRES) / (1 + (f(A_stat) * EquipRES)))
    
- **디버프 상태 이상 누적 공식 (Stack):** Stack_turn = Stack_current + Math.floor(Tick * (1 - RES)) - Recovery _(※ 누적치가 100에 도달 시 물리 오염(방어력 반토막) 또는 연산 오염(RAM 0 고정) 페널티 발동)_
    

## 6. 전투 로그 출력 가이드라인 (표기 정규화)

텍스트 RPG의 가독성을 보장하고 긴 숫자로 인한 피로감을 제거하기 위한 텍스트 인코딩 규칙입니다.

- **대미지 출력 규칙:** 십만 배수 스케일링으로 인해 숫자가 천만·억 단위를 넘을 경우, 아라비아 숫자와 한글 단위를 혼용하여 출력합니다.
    
    - _예시:_ "당신은 기계 괴수의 장갑판을 강타하여 **4,520만**의 대미지를 입혔습니다."
        
    - _예시:_ "프로토콜 브레이커가 현실 코드를 왜곡하여 **1억 2,840만**의 치명적 대미지를 꽂았습니다."
        
- **체력 및 자원 출력 규칙:** 백배수로 인해 만 단위 내외로 통제되므로, 가시성을 위해 천 단위 쉼표(`,`) 포맷을 상시 유지합니다.
    
    - _예시:_ "[위험] 적의 반격으로 HP가 **12,400** 감소했습니다. (남은 생명력: 32,600 / 65,000)"
