## 🛠️ 텍스트 RPG 개발의 3대 핵심 뼈대

### 1. 플레이어 상태 (State)

게임이 켜져 있는 동안 실시간으로 변하는 유저의 데이터(체력, 허기, 전직 가중치 등)를 담아두는 바구니입니다.

Python

```
# 게임 시작 시 메모리에 생성되는 플레이어 데이터
player = {
    "hp": 1500,
    "max_hp": 1500,
    "hunger": 100,
    "thirst": 100,
    "equipment": {"weapon": "none", "armor": "none"},
    # Phase 2에서 사용할 성향 가중치 배열
    "weights": {
        "kinetic": 0,
        "scrap": 0,
        "cyber": 0
    }
}
```

### 2. 이벤트 데이터 (Data)

방금 제가 정리해 드린 JSON 구조의 스크립트입니다. 이건 코드와 분리해서 파일로 보관합니다.

Python

```
# 기획서 내용을 데이터화한 것 (DB나 JSON 파일로 관리)
events = {
    "ACT1_SESSION_01": {
        "text": "무너진 거대 정찰 드론 잔해 밑에 쓸만한 부품이 보입니다.",
        "choices": [
            {"id": 1, "text": "완력으로 파이프를 뽑는다.", "weight": "kinetic", "reward": "WEAPON_SCRAP_01"},
            {"id": 2, "text": "배선을 정밀 해체한다.", "weight": "scrap", "reward": "PART_SCRAP_01"},
            {"id": 3, "text": "사이버덱으로 백도어를 심는다.", "weight": "cyber", "reward": "PART_SCRAP_02"}
        ]
    }
}
```

### 3. 게임 메인 루프 (Engine)

유저에게 **글을 보여주고 ➔ 번호를 입력받고 ➔ 상태를 업데이트**하는 무한 반복문입니다.

Python

```
def play_event(event_id):
    event = events[event_id]
    
    # 1. 스크립트 출력
    print(event["text"]) 
    
    # 2. 선택지 화면에 뿌리기
    for choice in event["choices"]:
        print(f"[{choice['id']}] {choice['text']}")
        
    # 3. 유저 입력 받기
    user_input = int(input("행동을 선택하세요: ")) - 1
    selected = event["choices"][user_input]
    
    # 4. 핵심 연산 시스템 연동 (가중치 누적 및 보상 지급)
    chosen_weight = selected["weight"]
    player["weights"][chosen_weight] += 1  # 가중치 +1
    player["equipment"]["weapon"] = selected["reward"] # 아이템 장착
    
    print(f"\n-> 시스템 연산: {chosen_weight} 가중치 누적. {selected['reward']} 획득!")
```

## 🚀 개발을 시작하는 현실적인 3단계 로드맵

도대체 뭐부터 타이핑해야 할지 모르겠다면, 기획서 전체를 다 구현하려 하지 마시고 딱 이 순서대로만 프로토타입을 만드세요.

### 1단계: 콘솔창에 텍스트 띄우고 선택하기

- 화면에 텍스트 RPG 도입부 문장을 띄우고, 유저가 `1번`이나 `2번`을 입력하면 다음 문장으로 넘어가는 기본 입출력 기능부터 만듭니다. (이게 게임의 화면이 됩니다.)
    

### 2단계: 턴당 자원 감소 수식 넣기

- 유저가 선택지를 고를 때마다 `player["hunger"] -= 5` 같은 연산을 실행하게 하고, 이게 `0`이 되면 화면에 `[WARNING]` 글리치 경보가 뜨는 예외 처리를 붙입니다.
    

### 3단계: 가중치 정산 함수 만들기

- 보스전이 끝났을 때 `player["weights"]`에 쌓인 점수 중 가장 높은 값을 판별하여 컴뱃, 메카, 넷 포스 중 하나의 엔딩 스크립트를 호출하는 조건문(`if/elif`)을 작성합니다.
